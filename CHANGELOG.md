## [2.0.0] - 2026-04-07

### ✨ 架构级重构（不向下兼容）

#### 工程架构
- **BREAKING CHANGE**: 前后端目录彻底隔离，拆分 `backend/` + `frontend/`
- **BREAKING CHANGE**: Monorepo 架构，前后端独立 `package.json`
- **新增**: Docker 多阶段构建，非 root 用户运行
- **新增**: ESLint + Prettier 代码规范全量覆盖
- **新增**: 全套 `npm run` 命令体系（12个标准化命令）
- **优化**: 所有脚本归入 `scripts/` 目录，配套说明文档

#### 前端架构
- **BREAKING CHANGE**: 前端模块化 5 层分层架构
  - `components/` - UI 组件层
  - `services/` - 通信服务层
  - `utils/` - 通用工具层
  - `config/` - 配置常量层
  - `styles/` - 样式层
- **新增**: EventEmitter 事件总线，彻底解决 RoomManager 回调反模式
- **新增**: 全量参数校验 + 用户友好错误提示
- **新增**: Socket 请求超时 + 自动重连机制
- **移除**: 所有硬编码魔法值，统一常量管理

#### 后端架构
- **新增**: 统一错误码体系（22个标准错误码）
- **新增**: 参数校验中间件（房间/用户参数合法校验）
- **新增**: 标准化响应格式 `{success, code, message, data}`

- **新增**: PowerShell 全功能部署脚本 `deploy.ps1`
- **新增**: 失败自动回滚机制
- **新增**: 部署前权限 + 环境检查
- **新增**: 部署后状态自动验证

## [1.3.0] - 2026-04-07

### ✨ 数据库规范化设计

#### 三线表设计
- **新增**: 7 张核心表规范化设计
  - `t_user` - 用户表（16字段）
  - `t_room` - 房间表（13字段）
  - `t_room_player` - 房间玩家关联表（9字段）
  - `t_game_record` - 游戏对局记录表（8字段）
  - `t_game_detail` - 玩家对局详情表（7字段）
  - `t_room_message` - 房间聊天消息表（8字段）
  - `t_ranking_global` - 全局排行榜表（5字段）

#### 优化点
- 统一使用 `BIGINT` 主键支持海量数据
- 全量字段 `COMMENT` 注释
- 高频查询字段索引优化
- 外键级联删除保障数据一致性
- utf8mb4 字符集支持完整 emoji
